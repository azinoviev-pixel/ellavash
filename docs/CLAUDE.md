## ТРИ КРИТИЧНЫХ ПРАВИЛА — НЕ НАРУШАТЬ

### 1. БЕЗ ЭМОДЗИ. НИГДЕ.

Запрещено использовать эмодзи в любом виде:
- В HTML (заголовки, бейджи, иконки, кнопки)
- В Markdown (списки, секции, заметки)
- В CSV / XLSX / TXT
- В именах файлов
- В комментариях кода
- В чате с пользователем

**Чем заменять:**
- HTML → inline SVG (рисовать самому или брать из Lucide/Heroicons/Phosphor как SVG-разметку)
- Markdown/CSV → текстовые метки: `[!]`, `[STOP]`, `[CRIT]`, `[OK]`, `[v]`, `[i]`, `[X]`
- Допустимы: типографика (`•`, `→`, `←`, `↑`, `↓`, `▸`, `◆`, `■`, `§`, `¶`), математические (`+`, `−`, `×`, `÷`, `±`, `≈`, `≠`, `≤`, `≥`), кавычки/тире (`«»`, `„"`, `—`, `–`)

Перед сдачей любого файла — `grep` на эмодзи. Должно быть **0 совпадений**.

### 2. МОБИЛЬНАЯ ПРОВЕРКА. КАЖДЫЙ HTML.

Каждый HTML файл, который **попадает на сайт** (ellavash.es/docs/ или любую другую публикацию) или передаётся клиентке — обязан **корректно работать на мобильном телефоне**. Ничего не должно «плыть», переполнять экран, обрезаться.

**Чек-лист — выполнять КАЖДЫЙ РАЗ перед публикацией:**

1. [ ] `<meta name="viewport" content="width=device-width, initial-scale=1.0">` — в `<head>`
2. [ ] Все сетки `display: grid` имеют `@media (max-width: 800px) { grid-template-columns: 1fr; }`
3. [ ] Базовый font ≥ 15px на мобильном (используй `clamp()` или явные мобильные размеры)
4. [ ] Заголовки уменьшаются на мобильном (`h1` ~ 32-40px, не 64px+)
5. [ ] Нет горизонтального скролла на 375px вьюпорте (iPhone SE)
6. [ ] Тапы (кнопки/ссылки) — минимум 44×44 px
7. [ ] Таблицы — `overflow-x: auto` на контейнере или конвертация в карточки
8. [ ] Картинки — `max-width: 100%; height: auto;`
9. [ ] Footer/header не обрезаются на iPhone safe-area
10. [ ] Hero-блоки `grid-template-columns: 180px 1fr` → `1fr` на мобильном
11. [ ] Padding контейнеров уменьшается на мобильном (32px → 18px)

**Проверка перед деплоем:**
- Открыть в DevTools мобильный вьюпорт (375px iPhone SE, 412px Android)
- При возможности — открыть прод-URL на телефоне после деплоя
- Скриншот мобильной версии — приложить к выдаче пользователю (если уместно)

**ЗАПРЕЩЕНО заливать HTML на сайт без mobile-проверки**, даже если десктоп выглядит идеально.

### 3. ВЕРСИОНИРОВАНИЕ — НЕ ТРОГАТЬ СТАРЫЕ ФАЙЛЫ.

Файлы в проекте **никогда не редактируются на месте**. При любом изменении — создаётся новая версия с суффиксом `_v2`, `_v3`, `_v4` и т.д.

Старые версии остаются как история — пользователь должен иметь возможность откатиться.

Применяется ко всем форматам: `.html`, `.md`, `.xlsx`, `.csv`, `.txt`.
