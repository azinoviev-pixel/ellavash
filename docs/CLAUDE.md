## ТРИ КРИТИЧНЫХ ПРАВИЛА — НЕ НАРУШАТЬ

### 1. БЕЗ ЭМОДЗИ. НИГДЕ.

Запрещено использовать эмодзи в любом виде:
- В HTML (заголовки, бейджи, иконки, кнопки)
- В Markdown (списки, секции, заметки)
- В CSV / XLSX / TXT
- В именах файлов
- В комментариях кода
- В чате с пользователем

**Чем заменять:**
- HTML → inline SVG (рисовать самому или брать из Lucide/Heroicons/Phosphor как SVG-разметку)
- Markdown/CSV → текстовые метки: `[!]`, `[STOP]`, `[CRIT]`, `[OK]`, `[v]`, `[i]`, `[X]`
- Допустимы: типографика (`•`, `→`, `←`, `↑`, `↓`, `▸`, `◆`, `■`, `§`, `¶`), математические (`+`, `−`, `×`, `÷`, `±`, `≈`, `≠`, `≤`, `≥`), кавычки/тире (`«»`, `„"`, `—`, `–`)

Перед сдачей любого файла — `grep` на эмодзи. Должно быть **0 совпадений**.

### 2. МОБИЛЬНАЯ ПРОВЕРКА. КАЖДЫЙ HTML.

Каждый HTML файл, который **попадает на сайт** (ellavash.es/docs/ или любую другую публикацию) или передаётся клиентке — обязан **корректно работать на мобильном телефоне**. Ничего не должно «плыть», переполнять экран, обрезаться.

**Чек-лист — выполнять КАЖДЫЙ РАЗ перед публикацией:**

1. [ ] `<meta name="viewport" content="width=device-width, initial-scale=1.0">` — в `<head>`
2. [ ] Все сетки `display: grid` имеют `@media (max-width: 800px) { grid-template-columns: 1fr; }`
3. [ ] Базовый font ≥ 15px на мобильном (используй `clamp()` или явные мобильные размеры)
4. [ ] Заголовки уменьшаются на мобильном (`h1` ~ 32-40px, не 64px+)
5. [ ] Нет горизонтального скролла на 375px вьюпорте (iPhone SE)
6. [ ] Тапы (кнопки/ссылки) — минимум 44×44 px
7. [ ] Таблицы — `overflow-x: auto` на контейнере или конвертация в карточки
8. [ ] Картинки — `max-width: 100%; height: auto;`
9. [ ] Footer/header не обрезаются на iPhone safe-area
10. [ ] Hero-блоки `grid-template-columns: 180px 1fr` → `1fr` на мобильном
11. [ ] Padding контейнеров уменьшается на мобильном (32px → 18px)

**Проверка перед деплоем:**
- Открыть в DevTools мобильный вьюпорт (375px iPhone SE, 412px Android)
- При возможности — открыть прод-URL на телефоне после деплоя
- Скриншот мобильной версии — приложить к выдаче пользователю (если уместно)

**ЗАПРЕЩЕНО заливать HTML на сайт без mobile-проверки**, даже если десктоп выглядит идеально.

### 3. ВЕРСИОНИРОВАНИЕ — НЕ ТРОГАТЬ СТАРЫЕ ФАЙЛЫ.

## 4. ЗАПРЕЩЕНО упоминать AI / методологию в документах

В любых документах проекта (HTML / Markdown / CSV / XLSX), которые видит Элен или клиенты, **запрещены** упоминания:
- «deep research», «глубокий research», «research v2» как маркер AI-методологии
- «Critical Thinker», «critical thinker» — это название моей внутренней роли
- «Reviewer», «Verifier», «Double Checker», «verifier-агенты», «reviewer-агенты»
- «AI», «ИИ», «agent», «агент» в значении AI-помощника
- «галлюцинации», «верификация на галлюцинации»
- Любые отсылки к «8 параллельных агентов», «фан-аут поиск», «adversarial verification»

**Чем заменять:**
- «deep research» → «детальный анализ рынка» / «исследование»
- «Critical Thinker анализ» → «анализ реалистичности» / «оценка рисков»
- «verifier-агенты» → «перепроверка по источникам» / «cross-verification»
- «8 параллельных агентов» → «cross-проверка по 8 направлениям»
- «AI Overviews» (как технология Google) — допустимо в контексте SEO

**Почему:** документы попадают к Элен и потенциально к клиентам/дистрибьюторам. Они должны выглядеть как нормальный аналитический проект, а не «AI-генерация». Внутренние методологические термины — только в моих заметках, не в финальных артефактах.

## 5. КАЖДОЕ упоминание компании → URL

При первом упоминании любой компании в HTML/Markdown — давать прямую ссылку на её сайт. Чтобы Элен могла кликнуть и проверить.

Пример: `<a href="https://www.bonpreuesclat.cat">Bonpreu / Esclat</a>` или в MD `[Bonpreu / Esclat](https://www.bonpreuesclat.cat)`.

Если URL неизвестен — пометить `[требуется уточнение URL]` явно, не молчать.

## 6. КАЖДАЯ цена / цифра → источник

При указании любой цены, оборота, доли рынка, процента роста — в скобках или сноской указывать источник.

Примеры:
- «Hacendado Pita 400 г — €1,56 ([Mercadona.es, май 2026](https://www.mercadona.es))»
- «Frozen bakery CAGR 8,75% ([Deep Market Insights](https://deepmarketinsights.com/vista/insights/frozen-bakery-product-market/spain))»

Если источник не уверен — пометить `[оценка]` или `[требует подтверждения]`.

Это правило обязательно для:
- Финансовой модели (каждая статья расходов)
- Анализа рынка (каждая цифра)
- Конкурентного анализа (обороты, доли, описания)
- Стратегии продаж (цены конкурентов)
