Раздел 7. Рекомендация по hreflang-стратегии

Вариант A: один сайт ellavash.es с языковыми версиями (рекомендуется)

ellavash.es/         → испанский (главный)
ellavash.es/en/      → английский (UK + intl)
ellavash.es/fr/      → французский
ellavash.es/de/      → немецкий
ellavash.es/it/      → итальянский (Фаза 4)

С корректным <link rel="alternate" hreflang="..." href="..."> в каждом HTML.

Плюсы:
- Один домен → накопление domain authority быстрее
- Дешевле в поддержке (1 хостинг, 1 SSL, 1 GitHub repo)
- Понятная архитектура для GitHub Pages

Минусы:
- Один URL для всех стран — Google иногда показывает «не локальный» сайт
- Сложнее делать локальные цены/доставку

Вариант B: отдельные ccTLD (ellavash.fr, ellavash.de, ellavash.co.uk)

Плюсы:
- Сильнейший локальный SEO-сигнал
- Local pricing, local checkout, local trust

Минусы:
- 4× затрат на domain + hosting + maintenance
- Каждый сайт стартует с DA=0
- Сложно держать консистентность контента

Итог: на старте Вариант A. При обороте >€500K в стране и подтверждённом product-market fit — переезд на ccTLD.

Обязательные технические условия для Варианта A:
- hreflang="es-ES", hreflang="fr-FR", hreflang="de-DE", hreflang="en-GB", hreflang="x-default"
- Canonical URL для каждой языковой версии
- Sitemap.xml с указанием всех языковых версий
- Локализованные meta title/description (НЕ автоперевод, ручной перевод)
- Локальная валюта и контактный телефон на каждой версии
- Schema.org Organization с areaServed для каждой страны
